## ขั้นตอนที่ 0: เตรียมพร้อม

อ่านไฟล์ต่อไปนี้ก่อนเสมอ:
- `~/.hermes/context/psf_guidelines.md`
- `~/.hermes/context/thpsf_framework.md`

จากนั้นตรวจสอบ working directory:
- ต้องมี `PSF_part3.md` — ถ้าไม่มีให้แจ้งว่า "ยังไม่มี draft ที่จะ revise กรุณารัน /psf-writer ก่อนครับ"
- ตรวจหา version ล่าสุด: ถ้ามี `PSF_part3_v2.md`, `PSF_part3_v3.md` ฯลฯ ให้อ่าน version ล่าสุดแทน
- ถามอาจารย์ว่ายื่น **ระดับใด** (1/2/3) ถ้ายังไม่ทราบ

## ขั้นตอนที่ 1: Diagnostic Analysis

อ่าน draft ทั้งหมดแล้ว **แสดง diagnostic report** ก่อนทำการแก้ไขใด ๆ

### 1a. Dimension Coverage Map

ทำตารางแสดงว่า dimension ร/ส/ค แต่ละตัวมีหรือขาด:

```
| Dimension | กรณีที่ 1 | กรณีที่ 2 | หมายเหตุ |
|-----------|----------|----------|----------|
| ร x.x.x   | ✓ / ✗    | ✓ / ✗    |          |
| ส x.x.x   | ...      | ...      |          |
| ค x.x.x   | ...      | ...      |          |
```

ระบุเฉพาะ dimension ที่เกี่ยวข้องกับระดับที่ยื่น — ไม่ต้องแสดง dimension ทั้งหมดของ framework

### 1b. Reflective Depth Audit

สแกนทีละย่อหน้าแล้วจัดกลุ่มตามปัญหาที่พบ:

**🔴 Method-only paragraphs** — อธิบายแค่ "ทำอะไร" ไม่มี pedagogical finding
> ตัวอย่าง: "ใช้ OCR แปลงข้อสอบแล้วนำมาวิเคราะห์ด้วย SHAP" — ยังขาด: ค้นพบอะไร / นักศึกษาเข้าใจหรือ misconception เรื่องใด

**🟡 Weak So What** — มีผล แต่ยังเป็น generalization ไม่มีหลักฐานเชิงตัวเลขหรือเนื้อหาที่เฉพาะเจาะจง
> ตัวอย่าง: "นักศึกษาเข้าใจมากขึ้น" — ควรระบุ: กลุ่มไหน เท่าไหร่ เมื่อเทียบกับอะไร

**🟡 Missing Now What** — เล่าว่าค้นพบอะไร แต่ไม่ได้บอกว่าปรับการสอนอย่างไรหลังจากนั้น

**⚪ Missing limitation** — ไม่มีการยอมรับข้อจำกัดเลย ทำให้เอกสารดู overclaim

### 1c. Level-Specific Gap Check

ตรวจตาม **ระดับที่ยื่น** เท่านั้น:

**ระดับ 1:** ครบ 4 ด้านในแต่ละกรณีศึกษา + มี evidence อย่างน้อย 1 รายการต่อด้าน

**ระดับ 2 (เพิ่มจากระดับ 1):**
- [ ] มีหลักฐานการแลกเปลี่ยน/ช่วยเพื่อนอาจารย์อย่างเป็นรูปธรรม
- [ ] มีนวัตกรรมการสอนที่ชัดเจน พร้อมผลที่วัดได้
- [ ] มีการประเมินและปรับปรุงตนเองอย่างเป็นระบบ

**ระดับ 3 (เพิ่มจากระดับ 2):**
- [ ] วิจัยในชั้นเรียน — มีผลลัพธ์เชิงเนื้อหา ไม่ใช่แค่บอกว่าทำวิจัย
- [ ] นวัตกรรมการประเมิน — แยกให้ชัดจากนวัตกรรมการสอน
- [ ] Impact ระดับหลักสูตรหรือองค์กร — แสดงว่าเกินกว่าห้องเรียนตัวเอง
- [ ] บทบาทที่ปรึกษา/พี่เลี้ยงอาจารย์ในองค์กร
- [ ] ความเป็นผู้นำด้านจรรยาบรรณ

### 1d. สรุปและขอ confirm

แจ้งอาจารย์ด้วยรายการ **สิ่งที่ดีแล้ว** และ **สิ่งที่ควรปรับปรุง** พร้อม priority:

> "วิเคราะห์แล้วพบสิ่งที่ควรปรับปรุง X จุดครับ จะขอแก้ไขตามนี้:
> [รายการ priority สูง]
> ต้องการให้แก้ทั้งหมด หรือเลือกเฉพาะจุดที่ต้องการได้เลยครับ"

**รอ confirm ก่อน — ห้ามแก้ไขโดยไม่ได้รับการยืนยัน**

## ขั้นตอนที่ 2: Revision

### 2a. กำหนด output filename

ตรวจจำนวน version ในโฟลเดอร์:
```bash
ls PSF_part3_v*.md 2>/dev/null | sort -V | tail -1
```
- ถ้ายังไม่มี → ใช้ `PSF_part3_v2.md`
- ถ้ามีแล้ว → increment เลข (v2 → v3 → v4)

### 2b. กฎการแก้ไข

- **ห้ามแตะต้นฉบับ** (`PSF_part3.md` หรือ version ก่อนหน้า)
- แก้เฉพาะจุดที่อาจารย์ confirm เท่านั้น
- รักษา voice ของอาจารย์ (`ข้าพเจ้า...`) — ห้ามเขียนทับทั้งย่อหน้าถ้าย่อหน้านั้นยังดีอยู่
- ใช้ข้อมูลจาก `teaching_cases.md` เท่านั้น ห้ามแต่งข้อมูลที่ไม่มี

### 2c. หลักการแก้ไข Pedagogical Reasoning

สำหรับแต่ละ method-only paragraph ที่ต้องแก้ ให้เพิ่มเนื้อหาในลำดับนี้:

**เพิ่ม pedagogical finding:**
> อธิบายว่า *ค้นพบอะไร* จากการใช้เครื่องมือหรือวิธีการนั้น
> ตัวอย่าง: "จากการวิเคราะห์พบว่านักศึกษากลุ่มที่มีคะแนนต่ำกว่า 50% มีรูปแบบการตอบคำถามที่แสดง misconception เรื่อง X อย่างชัดเจน"

**เชื่อมกับ educational principle:**
> ระบุว่าสิ่งที่ค้นพบสอดคล้องหรือขัดแย้งกับแนวคิดอะไร
> ไม่ต้องใส่ citation ทุกครั้ง — ใส่เมื่อเสริมความน่าเชื่อถือจริง ๆ

**เพิ่ม limitation (อย่างน้อย 1 จุดต่อกรณีศึกษา):**
> ยอมรับข้อจำกัดของวิธีการหรือผลที่ได้ อย่างตรงไปตรงมา
> ตัวอย่าง: "อย่างไรก็ตาม ข้อจำกัดของการใช้ OCR กับลายมือนักศึกษาทำให้ accuracy ของการแปลงข้อมูลยังไม่สมบูรณ์..."

**เพิ่ม Now What ที่เฉพาะเจาะจง:**
> บอกว่า *เปลี่ยนแปลงอะไรจริง ๆ* ในการสอนถัดไป ไม่ใช่แค่ "จะพัฒนาต่อ"
> ตัวอย่าง: "ในเทอมถัดไป ข้าพเจ้าปรับลำดับการสอน concept X ให้มาก่อน concept Y..."

### 2d. บันทึกและ render

บันทึก version ใหม่ จากนั้น render:
```bash
pandoc PSF_part3_v[N].md -o PSF_part3_v[N].docx \
  --reference-doc=$HOME/.hermes/psf_template.docx
```

## ขั้นตอนที่ 3: Diff Summary

หลัง render แสดงสรุปสิ่งที่แก้ไขในรูปแบบ:

```
📝 สรุปการแก้ไข: PSF_part3.md → PSF_part3_v[N].md

แก้ไขแล้ว:
✓ [ชื่อย่อหน้า/section] — เพิ่ม pedagogical finding เรื่อง X
✓ [ชื่อย่อหน้า/section] — เพิ่ม limitation ของ Y
✓ [ชื่อย่อหน้า/section] — ขยาย Now What ให้ระบุการเปลี่ยนแปลงการสอนที่เฉพาะเจาะจง

ไม่ได้แก้:
- [section] — ดีอยู่แล้ว ไม่จำเป็นต้องแก้

ไฟล์ที่มีตอนนี้:
- PSF_part3.md (ต้นฉบับ)
- PSF_part3_v[N].md (version ล่าสุด)
- PSF_part3_v[N].docx (สำหรับตรวจสอบ)
```

## Revision Loop

```
render → แจ้งอาจารย์ตรวจสอบ → รอ comment
  ├── มี comment → แก้เฉพาะจุดที่ระบุ → increment version → re-render → วนซ้ำ
  └── "ผ่าน" / "ตกลง" → แจ้งว่า version ไหนเป็น final และจะนำไปใช้ใน PSF_Application
```

เมื่ออาจารย์ confirm version final แล้ว ให้ render ไฟล์รวม:
```bash
pandoc PSF_part2.md PSF_part3_v[N].md -o PSF_Application.docx \
  --reference-doc=$HOME/.hermes/psf_template.docx

pandoc PSF_part2.md PSF_part3_v[N].md -o PSF_Application.pdf \
  --pdf-engine=xelatex \
  --include-in-header=$HOME/.hermes/thai_pdf.tex
```
