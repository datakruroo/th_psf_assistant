   - ข้อมูลส่วนตัว / ประวัติการศึกษา / ภาระงานสอน

   - หลักฐานที่ระบุไว้และสถานะของแต่ละชิ้น

2. **จัดเรียงลงใน working copy** โดยใช้โครงสร้างเดียวกับ `/input/cv_background.md` และ `/input/teaching_cases.md`

   - ข้อมูลที่คลุมเครือ → ใส่ไว้พร้อม comment `<!-- ตรวจสอบ: ... -->`

## ขั้นตอนที่ 0b: Intake (กรณียังไม่มีข้อมูล)

ยินดีต้อนรับอาจารย์ พร้อมแจ้งว่าระบบจะถามข้อมูลทีละกลุ่ม ได้แก่ ข้อมูลส่วนตัว ประวัติการศึกษาและภาระงานสอน กรณีศึกษาที่ 1 กรณีศึกษาที่ 2 ผลงานผู้เรียน บทบาทองค์กร เส้นทางอาชีพ รางวัล/จรรยาบรรณ และการดูแลผู้เรียน

**กลุ่ม 1 — ข้อมูลส่วนตัว**

ถาม: ชื่อ-นามสกุล (ไทย/อังกฤษ) / ตำแหน่งทางวิชาการ / คณะ-สถาบัน / สาขาวิชาหลักที่สอน / ระดับ PSF ที่ต้องการยื่น (1/2/3/4) / เคยยื่นมาก่อนหรือไม่

รอรับคำตอบก่อนถามกลุ่มถัดไป

**กลุ่ม 2 — ประวัติการศึกษาและภาระงานสอน**

ถาม: วุฒิการศึกษาทุกระดับ (สาขา มหาวิทยาลัย ปีสำเร็จ เรียงจากสูงสุด) / ภาระงานสอน 3 ปีการศึกษาล่าสุด (รหัสวิชา ชื่อวิชา ระดับ ชั่วโมงทฤษฎี ชั่วโมงปฏิบัติ)

**กลุ่ม 3 — กรณีศึกษาที่ 1**

ถามตามลำดับ รอคำตอบแต่ละข้อก่อนถามต่อ:

1. **ข้อมูลรายวิชา:** ชื่อ รหัส ระดับ จำนวนนิสิต ปีการศึกษา เป้าหมายหลัก และ CLO ทุกข้อพร้อมข้อความเต็ม
2. **ปัญหาก่อนออกแบบ:** "ก่อนออกแบบหรือปรับปรุงวิชานี้ พบปัญหาหรือความท้าทายอะไรบ้างครับ"
3. **ด้านที่ 1 — ออกแบบรายวิชา:** กรอบ/หลักการ ลำดับการเรียนรู้ นวัตกรรม หลักฐานที่มี (ชื่อเอกสาร + สถานะ: พร้อมใช้/คำบอกเล่า/ควรมีเสริม)
4. **ด้านที่ 2 — จัดการเรียนรู้:** กิจกรรมจริง วิธีสร้างการมีส่วนร่วม หลักฐานที่มี
5. **ด้านที่ 3 — วัดและประเมินผล:** สัดส่วนคะแนน วิธีให้ feedback หลักฐานที่มี สถิติผลการเรียนทั้งชั้น (ปี/n/mean/SD/ร้อยละ แยกตามองค์ประกอบถ้ามี) และคะแนนประเมินความพึงพอใจรายวิชา
6. **ด้านที่ 4 — พัฒนาและปรับปรุง:** สิ่งที่ปรับปรุงแล้ว / อยู่ระหว่างพัฒนา / ผลที่กล่าวได้อย่างระมัดระวัง / ข้อจำกัดของหลักฐาน

ถ้าคำตอบใด claim โดยไม่มีหลักฐาน → ถามต่อ: "มีเอกสาร ตัวเลข หรือผลประเมินประกอบมั้ยครับ สถานะพร้อมใช้หรือเป็นคำบอกเล่า"

**กลุ่ม 4 — กรณีศึกษาที่ 2**

ถามในรูปแบบเดียวกับกลุ่ม 3 ทุกขั้นตอน ต้องเป็นวิชาหรือบริบทที่แตกต่างจากกรณีที่ 1

**กลุ่ม 5 — ผลงานผู้เรียนและบทบาทระดับหลักสูตร/องค์กร**

ถาม: ผลงานผู้เรียน (conference, รางวัล, ตีพิมพ์) พร้อมชื่องาน ปี และสถานะเผยแพร่ สาธารณะ/ไม่สาธารณะ / บทบาทระดับหลักสูตรและองค์กร เช่น กรรมการหลักสูตร กรรมการสอบ วิทยากร ผู้ทรงคุณวุฒิ พี่เลี้ยงอาจารย์อื่น การสอนร่วมหรือวิจัยร่วม พร้อมปีและหลักฐาน

**กลุ่ม 6 — การพัฒนาวิชาชีพ เส้นทางอาชีพ และจรรยาบรรณ**

ถาม: อบรม สัมมนา ศึกษาดูงาน หรืองานวิจัยที่ติดตามในช่วง 3 ปีที่ผ่านมา / การช่วยเหลือเพื่อนอาจารย์ / feedback ที่ได้รับและนำมาพัฒนาตนเอง

ถามเพิ่ม: เส้นทางอาชีพและแรงบันดาลใจ (optional, ไม่จำกัดกรอบ 3 ปี) / รางวัล เครื่องราชอิสริยาภรณ์ การได้รับแต่งตั้งด้านจรรยาบรรณ ประวัติการปฏิบัติงาน / เรื่องเล่าการดูแลและช่วยเหลือผู้เรียน โดยแจ้งว่าจะเขียนแบบไม่ระบุชื่อจริง เว้นแต่เป็นข้อมูลสาธารณะแล้ว

**บันทึกและยืนยัน**

บันทึกลง `/workspace/cv_background_working.md` และ `/workspace/teaching_cases_working.md` แจ้งอาจารย์ตรวจสอบ รอ confirm → ไป **ขั้นตอนที่ 1**

## ขั้นตอนที่ 1b: เลือก theme การนำเสนอ Part 3

หลัง Evidence Pre-check ให้วิเคราะห์ข้อมูลที่มีจริง แล้วเสนอ 2–3 ทางเลือกพร้อมข้อดี-ข้อเสีย:

- **แบบ ก:** 2 กรณีน้ำหนักเท่ากัน เดินตาม 4 ด้าน (ออกแบบ/จัดการเรียนรู้/วัดผล/ปรับปรุง)
  - เหมาะเมื่อทั้งสองกรณีมีข้อมูลและหลักฐานใกล้เคียงกัน
  - ข้อควรระวัง: เสี่ยงอ่านเป็น checklist ถ้าไม่เชื่อม narrative
- **แบบ ข:** กรณีหลักเจาะลึกเดินเรื่องตาม CLO ทีละข้อ + กรณีที่สองกระชับกว่า
  - เหมาะเมื่อกรณีหลักมี CLO, สถิติผลการเรียน, feedback loop และผลงานผู้เรียนหนาแน่น
  - ข้อควรระวัง: ต้องยังแสดงกรณีที่สองให้เห็นชัดว่าเป็นอีกบริบทหนึ่ง
- **แบบ ค:** เดินเรื่องตามพัฒนาการ PDCA ข้ามภาคการศึกษา
  - เหมาะเมื่อมีข้อมูลก่อน–หลังหรือการปรับปรุงหลายรอบ
  - ข้อควรระวัง: ต้องไม่ทำให้ 2 กรณีศึกษาหายไป

ให้ agent แนะนำแบบที่เหมาะกับข้อมูลจริงพร้อมเหตุผล แต่อาจารย์เป็นผู้ตัดสินใจ ต้องรอคำตอบก่อนเขียน Part 2/3

ถ้ามีข้อมูลเส้นทางอาชีพและแรงบันดาลใจ ให้เสนอ **ย่อหน้าเปิดแบบ narrative** เป็นค่า default; ถ้าไม่มี ให้เสนอแบบกระชับ

## ขั้นตอนที่ 3: เขียน Part 3 (Reflective Writing)

### 3a. วิเคราะห์ก่อนเขียน

อ่าน `/workspace/teaching_cases_working.md` ถ้ามี มิฉะนั้นอ่าน `/input/teaching_cases.md` แล้ว mapping:
- dimension ร/ส/ค ที่เกี่ยวข้องกับแต่ละด้าน
- educational principles ที่สนับสนุน
- evidence ที่มีในไฟล์
- CLO ที่จะใช้เป็น anchor ถ้าเลือก theme แบบ CLO-driven
- ข้อมูลที่มีสถานะเผยแพร่สาธารณะ เพื่อกำหนดว่าจะเอ่ยชื่อผู้เรียนจริงได้หรือไม่

แสดง mapping ให้อาจารย์ยืนยันทิศทางก่อนเขียน

### 3b. เขียน reflective writing

เขียนตาม model **What–So What–Now What** ทุกย่อหน้า:
- **What:** อธิบายสิ่งที่ทำพร้อม context
- **So What:** วิเคราะห์เชิงวิพากษ์ — *ค้นพบอะไร* (ไม่ใช่แค่บอกว่าใช้เครื่องมืออะไร) พร้อมหลักฐาน เชื่อมกับ educational principle
- **Now What:** เปลี่ยนแปลงการสอนอย่างไรจริง ๆ — เฉพาะเจาะจง ไม่ใช่แค่ "จะพัฒนาต่อ"

**กฎที่ต้องทำทุกระดับ:**
- ทุก claim สำคัญมีรหัส **2 ระดับ** เป็น default เช่น **(ร 1.2)** / **(ส 2.4)** / **(ค 3.1)** กำกับท้ายประโยคหรือวรรค ความถี่ต้องสูง แทบทุก claim สำคัญต้องมีรหัส
- ใช้รหัส 3 ระดับ เช่น (ส 2.4.1) ได้เมื่ออาจารย์ต้องการความเฉพาะเจาะจง แต่ห้ามบังคับ 3 ระดับ
- Educational principles กระจายในเนื้อหา ไม่แยก section
- ใช้เฉพาะข้อมูลใน `/workspace/teaching_cases_working.md` หรือ `/input/teaching_cases.md` ห้ามแต่งเพิ่ม
- เขียน first person ("ข้าพเจ้า...")

## Revision Loop ทั่วไป

หลัง render แต่ละส่วน วน loop จนอาจารย์ยืนยัน:

```
render → แจ้งอาจารย์ตรวจสอบ → รอ comment
  ├── มี comment → แก้เฉพาะจุดที่ระบุ → re-render → วนซ้ำ
  └── "ผ่าน" / "ตกลง" / "โอเค" → ไปขั้นตอนถัดไป
```

- แก้เฉพาะจุดที่อาจารย์ระบุ ห้ามแตะส่วนอื่น
- Re-render ทันทีทุกครั้งที่แก้
- ถ้า comment ไม่ชัดเจน → ถามให้ชัดก่อนแก้
- ห้ามไปขั้นตอนถัดไปจนกว่าจะได้รับคำยืนยันชัดเจน
