### 0a. Welcome Message

แสดงข้อความต้อนรับต่อไปนี้ทันที (ห้ามย่อ ห้ามตัด):

---

**ยินดีต้อนรับสู่ระบบช่วยเขียน Thailand-PSF**

ก่อนเริ่มร่างเอกสาร ระบบจะขอรวบรวมข้อมูลจากอาจารย์ก่อน เพื่อให้ร่างที่ได้มีความถูกต้องและตรงกับประสบการณ์จริงของอาจารย์

**ข้อมูลที่ต้องเตรียม (5 กลุ่ม)**

| กลุ่ม | รายการ | สเปค |
|------|--------|------|
| **1. ข้อมูลส่วนตัว** | ชื่อ-นามสกุล (ไทย/อังกฤษ), ตำแหน่งทางวิชาการ, คณะ, สถาบัน, สาขาวิชาหลักที่สอน, email, เบอร์โทร | ใช้สำหรับหน้าปก PSF-01 |
| **2. ประวัติการศึกษา** | วุฒิการศึกษาทุกระดับ (สาขา มหาวิทยาลัย ปีสำเร็จ) | เรียงจากสูงสุดลงมา |
| **3. ภาระงานสอน 3 ปี** | ตาราง: ปีการศึกษา / รหัส-ชื่อวิชา / ระดับ / ชั่วโมงทฤษฎี / ชั่วโมงปฏิบัติ | ย้อนหลัง 3 ปีการศึกษา |
| **4. กรณีศึกษาที่ 1** | รายวิชาที่เลือก + ข้อมูล 4 ด้าน (ดูด้านล่าง) | วิชาที่สอนจริง ย้อนหลังไม่เกิน 3 ปี |
| **5. กรณีศึกษาที่ 2** | รายวิชาที่ต่างจากกรณีที่ 1 + ข้อมูล 4 ด้าน | ต้องต่างบริบทจากกรณีที่ 1 |

**ข้อมูล 4 ด้านในแต่ละกรณีศึกษา:**
- **ด้านที่ 1 — การออกแบบรายวิชา:** ออกแบบอย่างไร ใช้หลักการใด มีนวัตกรรมอะไร + หลักฐาน/ผลที่เกิด
- **ด้านที่ 2 — การจัดการเรียนรู้:** ดำเนินการสอนอย่างไร สร้างการมีส่วนร่วมอย่างไร + หลักฐาน/ผลที่เกิด
- **ด้านที่ 3 — การวัดและประเมินผล:** วิธีประเมิน วิธีให้ feedback + หลักฐาน/ผลที่เกิด
- **ด้านที่ 4 — การพัฒนาและปรับปรุง:** เรียนรู้อะไร ปรับปรุงอะไรแล้ว/วางแผนจะทำ

> **หมายเหตุ:** "หลักฐาน" หมายถึงข้อมูลเชิงประจักษ์ เช่น คะแนนประเมินรายวิชา ผลการเรียน ข้อมูลเชิงตัวเลข หรือคำพูดจากนิสิต — ยิ่งละเอียดยิ่งดี

ระบบจะถามข้อมูลทีละกลุ่ม อาจารย์ตอบได้เลย พิมพ์สั้นหรือยาวก็ได้ ระบบจะช่วยจัดรูปแบบให้

รอรับคำตอบ → บันทึกไว้ใน memory

### 0d. ถามข้อมูลกลุ่มที่ 3 — กรณีศึกษาที่ 1

> "ต่อไปคือหัวใจหลักของ PSF — กรณีศึกษาการสอน
> กรณีที่ 1: เลือกรายวิชาที่อาจารย์คิดว่าแสดงสมรรถนะด้านการสอนได้ดีที่สุดในช่วง 3 ปีที่ผ่านมา

> ขอทราบข้อมูลรายวิชาก่อน:

> - จำนวนนิสิต ปีการศึกษา/ภาคที่สอน
> - เป้าหมายหลักของวิชานี้"

รอรับคำตอบ จากนั้นถามทีละด้าน:

**ด้านที่ 1:** "อาจารย์ออกแบบรายวิชานี้อย่างไรครับ — ใช้หลักการอะไร จัดลำดับเนื้อหาอย่างไร มีกิจกรรมหรือนวัตกรรมอะไรที่ทำแตกต่างจากการสอนทั่วไป และมีผลหรือหลักฐานอะไรที่แสดงว่าได้ผลครับ"

รอรับคำตอบ ถ้าคำตอบสั้นหรือขาดหลักฐาน → ถามต่อ: "มีข้อมูลเชิงตัวเลขหรือผลประเมินประกอบมั้ยครับ เช่น คะแนน ความคิดเห็นนิสิต"

**ด้านที่ 2:** "การดำเนินการสอนจริงเป็นอย่างไรครับ — ใช้เทคนิคอะไร สร้างการมีส่วนร่วมอย่างไร มีปัญหาอะไรเกิดขึ้นและแก้ไขอย่างไร"

**ด้านที่ 3:** "วิธีการประเมินและให้ feedback เป็นอย่างไรครับ — ใช้ formative/summative อะไรบ้าง วิธีให้ feedback กับนิสิต และผลที่เกิดเป็นอย่างไร"

**ด้านที่ 4:** "จากการสอนวิชานี้ อาจารย์เรียนรู้อะไรครับ มีอะไรที่ปรับปรุงแล้วหรือวางแผนจะปรับปรุงบ้าง"

## ขั้นตอนที่ 3: เขียนส่วนที่ 3 — กรณีศึกษาที่ 1

### 3a. วิเคราะห์ก่อนเขียน

อ่าน `teaching_cases.md` กรณีที่ 1 แล้วทำ mapping:
- ระบุ dimension ร/ส/ค ที่เกี่ยวข้องกับแต่ละด้านของกรณีนี้
- ระบุ educational principles ที่ใช้สนับสนุน (เช่น Constructive Alignment, Bloom's Taxonomy)
- ระบุ evidence ที่มีอยู่ในไฟล์

แสดง mapping ให้อาจารย์เห็นก่อนเขียน เพื่อยืนยันทิศทาง

### 3b. เขียน reflective writing

เขียนตาม model What–So What–Now What:

**What:** อธิบายสิ่งที่ทำ (course design, กิจกรรม, การสอน, การประเมิน) พร้อม context ชัดเจน
**So What:** วิเคราะห์เชิงวิพากษ์ — ผลที่เกิด ทั้ง positive impact, ปัญหา, หลักฐาน
**Now What:** สิ่งที่เรียนรู้ แผนพัฒนา หรือการปรับปรุงที่ได้ทำแล้ว

กฎการเขียนทุกระดับ:
- ทุกย่อหน้าที่อ้างสมรรถนะต้องมี **(ร x.x.x)** หรือ **(ส x.x.x)** หรือ **(ค x.x.x)** กำกับ
- Educational principles ต้องกระจายตลอดเนื้อหา ไม่เขียนแยก section
- ใช้เฉพาะข้อมูลที่มีใน `teaching_cases.md` เท่านั้น ห้ามแต่งเพิ่ม
- เขียน first person ("ข้าพเจ้า...")

เพิ่มเติมตามระดับ:

**ระดับ 1** — เน้น 4 ด้านในห้องเรียนตัวเอง ไม่จำเป็นต้องกล่าวถึงเพื่อนอาจารย์หรือองค์กร

**ระดับ 2** — ต้องมีเนื้อหาครอบคลุม:
- การแลกเปลี่ยน/ช่วยเพื่อนอาจารย์ในสาขาอย่างเป็นรูปธรรม (ร 1.2.x, ส 2.x.x)
- นวัตกรรมหรือความคิดสร้างสรรค์ในการสอน (ส 2.2.x)
- การประเมินและปรับปรุงการสอนของตนเองอย่างเป็นระบบ (ส 2.4.x)

**ระดับ 3** — ต้องมีเนื้อหาครอบคลุมทั้งหมดของระดับ 2 และเพิ่ม:
- วิจัยในชั้นเรียนและนำผลมาพัฒนาการสอน (ส 3.1.x)
- นวัตกรรมการประเมิน — ไม่ใช่แค่นวัตกรรมการสอน (ส 3.3.x)
- Impact ระดับหลักสูตร — แสดงว่าการเปลี่ยนแปลงส่งผลเกินกว่าห้องเรียนตัวเอง (ส 3.2.x)
- บทบาทที่ปรึกษา/พี่เลี้ยงอาจารย์ในองค์กร (ร 3.x.x, ค 3.x.x)
- ความเป็นผู้นำด้านจรรยาบรรณในองค์กร (ค 3.2.x)

### 3c. ตรวจสอบก่อน render

- [ ] Word count ไม่เกินเกณฑ์ (นับรวมทั้งกรณีที่ 1 และ 2)
- [ ] ทุก claim มี dimension อ้างอิง
- [ ] มี evidence สนับสนุนอย่างน้อย 1 รายการต่อด้าน
- [ ] ไม่มีตาราง/รูปภาพในเนื้อหาหลัก

## ขั้นตอนที่ 6: ตรวจสอบ PSF-02 Checklist

หลัง approve ส่วนที่ 3 ครบแล้ว ตรวจสอบตาม **ระดับที่ยื่น**:

**ทุกระดับ — ตรวจ:**
- [ ] มี evidence อย่างน้อย 1 รายการต่อด้านต่อกรณีศึกษา
- [ ] ทุก claim มี dimension ร/ส/ค กำกับ
- [ ] ความยาวอยู่ในเกณฑ์ (ระดับ 1–2: ~3,000 คำ / ระดับ 3: ~5,000 คำ)

**ระดับ 2 — ตรวจเพิ่ม:**
- [ ] มีหลักฐานการแลกเปลี่ยนกับเพื่อนอาจารย์ในสาขา
- [ ] มีนวัตกรรมการสอนที่ชัดเจน
- [ ] มีการประเมินและปรับปรุงตนเองอย่างเป็นระบบ

**ระดับ 3 — ตรวจเพิ่ม:**
- [ ] มีหลักฐานวิจัยในชั้นเรียน (classroom research)
- [ ] มีนวัตกรรมการประเมิน (ไม่ใช่แค่นวัตกรรมการสอน)
- [ ] แสดง impact ระดับหลักสูตรหรือองค์กร
- [ ] มีบทบาทที่ปรึกษา/พี่เลี้ยงอาจารย์คนอื่นในองค์กร
- [ ] แสดงความเป็นผู้นำด้านจรรยาบรรณในองค์กร

ถ้าขาด item ใด → แจ้งอาจารย์ว่าต้องเพิ่มข้อมูลตรงไหน และถามว่ามีข้อมูลส่วนนั้นหรือไม่ก่อนแก้ไข

## Revision Loop (ทำซ้ำทุกครั้งหลัง render)

หลัง render แต่ละส่วนเสร็จ ให้รอ feedback และวน loop นี้จนอาจารย์ยืนยัน:

```
render → แจ้งอาจารย์ตรวจสอบ → รอ comment
  ├── มี comment → แก้เฉพาะจุดที่ระบุ → re-render → วนซ้ำ
  └── "ผ่าน" / "ตกลง" / "โอเค" → ไปขั้นตอนถัดไป
```

กฎการแก้ไข:
- แก้เฉพาะจุดที่อาจารย์ระบุ **ห้ามแตะส่วนอื่น**
- Re-render ทันทีทุกครั้งที่แก้
- แจ้งสั้น ๆ ว่าแก้อะไร ตรงไหน
- ถ้า comment ไม่ชัดเจน → ถามให้ชัดก่อนแก้
- **ห้ามไปขั้นตอนถัดไปจนกว่าจะได้รับคำยืนยันชัดเจน**
